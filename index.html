HTML och CSS för tillgänglighet

1. HTML element har en viktig roll

Problem

Informationen presenteras på ett inkorrekt sätt. Exempelvis kan informationen läsas upp fel av en skärmläsare, inte kunna förstoras eller presenteras med en annan layout. Viktig information kan gå förlorad.

Lösning

Ange i kod vad sidans olika delar har för roll. Detta är viktigt för att öka chansen att informationen presenteras på ett korrekt sätt. Detta görs genom att använda HTML elementen på rätt sätt. Exempel:

  • Skapa rubriker med h-element

    • <h1> h1 rubrik </h1>

  • Använd rätt HTML-element när listor görs

    • <ul> Lista utan numerering </ul>

  • Namnge formulärfält med label-elementet

    • <label> name </label>

2. Ange språk med HTML

Problem

När språket inte är angivet i koden riskerar skärmläsaren att inte presenta innehållet rätt, användaren missar automatisk översättning och kan få en inkorrekt avstavning.

Lösning

Ange sidans huvudspråk och språkförändringar i koden. För att göra textinnehåll mer läsbart och begripligt ska aktuellt språk anges i HTML-koden. Detta görs med en språkkod på sidans rot-element och ett lang-attribut.

<html lang=”sv”>

<div lang=”en”> This sentence is written in English </div>

3. Komplettera färger med HTML

Problem

Viss användare kan inte uppfatta färgskillnader och kan inte förlita sig på informaton som enbart förmedlas genom färg. Detta kan vara för att de använder skärmläsare, monokrom skärm eller är färgblinda.

Lösning

Med hjälp av CSS kan HTML element stylas med färger, men enbart färg ska inte användas för att förmedla information. Komplettera användandet av färg med:

  • Ikoner
  • Understrykning, fetstil eller annat teckensnitt
  • Skriven text som ger motsvarande information

4. Skapa tillräcklig kontrast med CSS

Problem

Personer med nedsatt syn har ofta svårt att läsa text med bristande kontrast mot textens bakgrund. Genom otillräcklig kontrast riskerar viktig information att missas.

Lösning

Använd tillräcklig kontrast mellan text och bakgrund. Genom CSS kan bakgrundsfärg och teckenfärg bestämmas. De flesta kan läsa brödtext på skärm om skillnaden i ljusintensitet mellan förgrund och bakgrund följer WCAG-standarden (som har förhållandet 4,5:1). Detta gäller även för ikoner, knappar och formulärfält.

.code-yellow {
font-family: "Courier New", Courier, monospace;
color: rgb(255, 251, 122);
background-color: rgb(43, 43, 43);
padding: 2%;
font-size: 16px;
}

5. Gör en logisk tab-ordning

Problem

Utan en logisk tab-ordning riskerar fokus att flytta till ett oväntat element. Det kan bli förvirrande och orsaka fel. Den som navigerar med tab-tangenten behöver också få veta tydligt var fokus ligger. Standardmarkeringen är ofta en tunn linje som är svår att se.

Lösning

För användare som navigerar med tangentbordet är det viktigt att fokus-ordningen är logisk i förhållande till hur innehållet presenteras på skärm eller i skärmläsare. Med hjälp av HTML och CSS kan markeringar göras tydliga och visar vilket element som ligger i fokus med hjälp av tangentbordet. Detta kan göras med en tabindex tag på HTML elementen. Markeringen kan göras ännu tydligare med hjälp av CSS.

h2:focus {
border: solid black 3px;
background-color: #ffff07;
padding: 2%;
}

6. Märk upp formulärfält i koden

Problem

Det blir svårare att fylla i formulär för användaren när vilket typ av inmatning som förväntas av användaren inte angetts i koden. Webbläsaren eller hjälpmedel föreslår heller inte automatisk inmatning.

Lösning

Ange i de olika HTML elementen vilken typ av innehåll som förväntas. Här kan attributet autocomplete användas på inmatningsfält.

7. Presentera innehållet i ordning

Problem

Elementen hamnar i oordinng och placering är beroende på skärm. Skärmläsare läser upp information i fel ordning.

Lösning

Presentera innehållet i ordning på ett meningsfullt sätt för alla. Genom till exempel genom CSS column attributet kan vi se till att informationen lägger sig rätt oavsett skärm.

.card {
background-color: rgb(243, 243, 243);
width: 50%;
height: 50%;
display: flex;
justify-content: center;
align-items: center;
flex-direction: column;
padding: 5%;
border-radius: 10px;
margin: 3%;
}

8. Skapa tillräckligt med avstånd mellan tecken, rader, stycken och ord

Problem

Det finns en risk är att innehåll krockar eller gömmer sig bakom annat innehåll när användaren vill påverka avståndet mellan rader, ord och tecken för att underlätta läsning.

Lösning

Placera text i utrymmen som är tillräckligt stora eller anpassar sig efter innehållet. Detta kan göras genom HTML. Ta hänsyn till att texten kan ändra storlek vid absolut positionering av element som riskerar att krocka med texten. Den relativa enheterna em, rem eller procent kan användas istället för pixlar.

html {
font-size: 125%; /*Motsvarar en storlek på 20px, default är 16px*/
}

p,
ul {
font-family: verdana;
font-size: 1rem; /* Motsvarar en storlek på 20px */
line-height: 1.5; /* WCAG guideline */
text-align: justify;
max-width: 65ch;
}

9. Skapa en flexibel layout

Problem

Layouter som inte fungerar vid förstoring eller liten skärm kan skapa problem för användaren som riskerar att missa viktig information. Detta kan också göra det extra svårt för personer med nedsatt synd. Att behöva scrolla i sidled kan också vara besvärligt och försämrar upplevelsen.

Lösning

Använd responsiv design. I CSS kan media queries skapa olika layouts beroende på skärmstorlek. Dessa media queries kan kombineras med andra metoder för att skapa flexibilitet mellan de olika brytpunkterna. Exempel är använda procent med CSS egenskaperna maximal och minimal bredd.

@media screen and (max-width: 992px) {
.card {
width: 90%;
}
